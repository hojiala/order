流水號模式

「依取餐時間」時，營業日開始前送出的線上預訂單先不配號也不列印；到流水號每日歸零時間後，店內主機立即把當日所有隔夜預訂單依取餐時間排序配號（06:00 → No.001）並依序列印。營業中新下的訂單照常即時配號、即時列印。

ℹ️ 批次配號時間＝POS「流水號設定」裡的重置時間（建議設在開店前，例如 05:30）。此模式需要店內主機 PocketBase 的 pb_hooks 為 2026-07-14 之後的版本，且主機與印表機 24 小時開機。

Node-RED 直接列印

填入店內 Node-RED 的列印 API 網址後，POS / 桌邊 / 線上點餐送單時會直接呼叫店內 Node-RED 列印，不再繞經 Firebase print_queue，可把出單延遲從 7 秒左右降到 1 秒。留空則沿用原本的 Firebase 列印流程。

⚠️ 若點餐頁是用 https:// 開啟，瀏覽器會封鎖對 http:// 內網位址的請求（混合內容），此時直接列印會失敗並自動退回 Firebase。要真正生效，請讓店內機器以 http:// 或內網網域開啟點餐頁，或改用可經 HTTPS 存取的 Node-RED 網址。

此網址只會保存在店面設定，供店內裝置呼叫；只有列印觸發改走這裡，訂單資料仍照常雙寫（PocketBase 主庫 → hook → Firebase 備援）。
